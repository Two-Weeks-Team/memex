A memory you already own.

You don't document every thought. Yet when the moment arrives, the relevant one surfaces on its own. Memex does that to your Claude Code / Codex session history: it stays latent until a context approaches it — then it comes back to you.

kind: associative memory over your transcripts runtime: no LLM, no telemetry store: Qdrant
Everyone else shipped a chatbot. We didn't.
  • Not a chat window — there's nothing to prompt.
  • Not a search box — you don't already know the keyword.
  • Not an LLM at runtime — nothing is generated, only recalled.

It's the idea from Vannevar Bush's 1945 Memex: an associative trail through things you've already thought. You move by similarity, not by query. The thought finds you.

latent · margin note Claude Code silently migrates and deletes old sessions. The corpus you build with it is not promised to you tomorrow. Memex keeps a copy you own — and lets you walk back through it.

what it is, plainly

A local tool that reads your session history — ~/.claude/projects/**/*.jsonl, legacy transcripts, and history.jsonl — and turns it into a navigable spatial / associative memory. It runs offline. Nothing is generated; everything is recalled by distance in vector space.

    an associative trail, not a query

    A query returns rows. A trail connects neighbors. Hover or focus a node to see the edges light up — the way one recalled thought pulls the next one with it.

    your agents can read it too

    Memex exposes 11 MCP tools over stdio + HTTP, so Claude Code, Codex, and Cursor can walk your memory the same way you do — recall by similarity, mid-task. 

    # register Memex as an MCP server in Claude Code
claude mcp add memex -- memex serve --stdio

# or run the all-in-one server variant (Qdrant + web + MCP in one image)
docker run -p 8080:8080 ghcr.io/two-weeks-team/memex

    11 tools, one corpus. The agent stops re-deriving what you already worked out last week.
