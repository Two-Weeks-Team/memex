why it feels involuntary

It isn't search. The situation calls the memory up.

A search box waits for you to know what you've forgotten — which, mid-incident at 3am, you don't. Memex works the other way around. The live context becomes a query you never typed, and the nearest pieces of your own history surface on their own cadence.

Context becomes the query

The error text, the failing service, the stack you're staring at — all of it is embedded and matched against your corpus by similarity. No keywords to guess.

Associative trails, not folders

Vannevar Bush's 1945 idea: move through memory by association. Qdrant holds the vectors; the trail from "now" to "that time before" is distance, not a filename.

On its own timing

Proactive Recall pushes the relevant fragment when the moment evokes it — irregular, lifelike, unbidden. You're the recipient, not the seeker.

the corpus you own

Your Claude Code & Codex history, made navigable.

Claude Code silently migrates and deletes old sessions. Memex reads ~/.claude/projects/**/*.jsonl (plus legacy transcripts and history.jsonl), embeds it into Qdrant, and gives you a handful of ways to move through it by similarity instead of keywords. No model runs at query time.

put it where you already work

Recall inside the agent — 11 MCP tools over stdio + HTTP.

The same recall runs as an MCP server, so your agent in Claude Code, Codex, or Cursor can pull from your past sessions while it works. A server/Docker variant bundles Qdrant + web UI + MCP into one image for a box you control.
