Not a chatbot. A memory you can move through.

Memex reads your existing Claude Code / Codex session history and turns it into a navigable spatial memory backed by Qdrant — so you travel by similarity, not keywords.

  • It's already yours. Reads ~/.claude/projects/**/*.jsonl, legacy transcripts, and history.jsonl — no new note-taking habit.
  • LLM-free at runtime. The recall layer runs on vectors in Qdrant, not a model answering questions.
  • Preserves what gets deleted. Claude Code silently migrates and removes old sessions; Memex keeps a corpus you own.
  • Inspired by Vannevar Bush's 1945 Memex — associative trails between fragments, not a single answer box.
  • Spatial by design. Distance between two memories means something: closer is more related.
  • Built for the Vector Space Day theme — thinking outside the bot, literally.

Six ways the field opens up

Each surface is a different way to walk the same associative space.

Time Machine

Replay a session as it unfolded — step back into the moment a decision was made.

Lens

Named-vector weighted search — tilt the field toward the kind of memory you want.

Topology

Distance-matrix to a minimum-spanning-tree graph — see the shape of how thoughts connect.

Mix / Discovery

Positive and negative recommendation — "more like this, less like that," then drift.

Proactive Recall

Relevant fragments surface on their own as your current context approaches them.

Companion · Wrapped · Loop Breaker

The agent-memory layer — dashboards, recaps, and nudges when you're going in circles.

Give your agent the memory it forgot

Memex exposes 11 MCP tools over stdio + HTTP. It works in Claude Code, Codex, and Cursor — and a server / Docker variant runs Qdrant, the web UI, and MCP together in one image.

$ claude mcp add memex -- memex serve --stdio
# or run the all-in-one server variant (Qdrant + web + MCP):
$ docker run -p 8080:8080 ghcr.io/two-weeks-team/memex
