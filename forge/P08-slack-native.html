Recall comes to you — not the other way around

Most tools wait for you to go looking. Memex watches the context you're already in and lets the relevant past rise — irregular, unprompted, the quiet click of "oh, right." It runs over your own Claude Code / Codex history, no model in the loop at runtime.

arrives unbidden

You don't query it

There's no search field and no chat prompt. A memory shows up because your current situation is close — by similarity — to a moment you've lived before.

similarity, not keywords

It travels by association

Vannevar Bush's 1945 Memex idea — associative trails between thoughts. Backed by Qdrant vectors over your transcripts, so "this feels like that" is the lookup.

a corpus you own

Nothing gets quietly deleted

Claude Code migrates and drops old sessions. Memex keeps the corpus (~/.claude/projects/**/*.jsonl + legacy + history.jsonl) yours.

Where it lives: inside the agent you're already using

The thread above is the idea, not a product chrome. In practice Memex rides along your coding agent through MCP — 11 tools over stdio + HTTP — so recall can surface where the work happens. A server/Docker variant bundles Qdrant + web + MCP in one image. 

# add Memex to Claude Code as an MCP server
claude mcp add memex -- memex serve --stdio

# or run the all-in-one server variant (Qdrant + web + MCP)
docker run -p 6333:6333 -p 8080:8080 \
  -v ~/.claude:/data/claude:ro \
  twoweeks/memex:server
