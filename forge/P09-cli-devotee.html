You didn't search for it.
The session printed itself into your scrollback.

You're mid-task in your terminal. A past session — the one where you already solved this — rises into the log on its own, because the moment called it up. No query typed. No box to fill. The history speaks up unbidden.

Memex turns your Claude Code / Codex session corpus into a navigable spatial memory backed by local Qdrant. The terminal below is a recreation of the proactive-recall watcher — recall arrives as an autonomous tail, not a grep you ran.

zsh — ~/dev/acme-saas — memex watcher attached recall: passive

The watcher polls your active session every 12s. Nothing here is typed by you — the recall block is the part that arrives unasked.

## recall as a tail, not a query

The trap is to build a search box. Memex's proactive layer is the opposite: while you work in one session, a 60-second watcher embeds your fresh turns and runs a filtered neighbor query against everything you've ever done. When a past session matches the situation you're in, it surfaces — to you, on its own cadence.

situation → not keystroke

Retrieval is triggered by the context you're in — a fresh tool_result.is_error, a repeated loop — not by something you typed. You are the recipient.

error vector + filter

A query() on the dedicated error named vector with a has_errors=true payload filter. The neighbor it finds is usually the session that resolved it.

zero LLM at runtime

No chatbot, no generative call in the loop. The "answer" is a vector neighbor in your own local Qdrant on localhost:6334. The corpus stays on your machine.

a corpus you own

Claude Code silently migrates and deletes old sessions. Memex reads both legacy and modern ~/.claude/projects/**/*.jsonl and snapshots them so cleanups can't reach them.

## install — terminal first

Memex is a desktop app (macOS, Apple Silicon, v0.1.0 — unsigned MVP) and a CLI over the same indexed corpus. Start Qdrant, index a corpus, then drive everything from the shell.

# 1. start Qdrant — the load-bearing dependency
bash scripts/start-qdrant.sh

# 2. index a corpus (synthetic sample ships with the repo — no private data)
memex scan --path examples/sample-corpus --index

# 3. drive the Qdrant-backed surfaces from the terminal
memex lens     "build error" --error 2.0 --content 1.0   # weighted named-vector lens
memex mix      --pos <id> --neg <id>                  # Discovery API recommendation
memex topology --sample 12 --out /tmp/topo.json       # Distance Matrix → MST
memex recall   "cargo build linker error"             # the proactive-recall query, by hand

Source build and clean-machine Gatekeeper steps: docs/INSTALL.md. Prefer headless? The server / Docker variant runs Qdrant + web UI/API + MCP in one image.

## mcp — your agent recalls through Memex

The same proactive recall, but for an AI agent. Wire Memex's MCP server into Claude Code once; then mid-conversation your agent can reach into your local session corpus — and the Cold Start Killer can inject a memory primer at turn 0, before you ask anything.

# one-time wiring — point Claude Code at the memex binary you already run
claude mcp add memex /path/to/memex/src-tauri/target/release/memex mcp

# …or let memex print the exact line for your machine
memex install-mcp            # echoes the `claude mcp add …` line
memex install-mcp --run      # runs it for you

claude mcp list

## what it actually is

  • A non-chatbot, LLM-free-at-runtime tool over your Claude Code / Codex session history.
  • Surfaces: Time Machine replay, Lens, Topology, Mix/Discovery, proactive Recall, plus the Companion / Wrapped / Loop Breaker agent-memory layer.
  • Built on Qdrant 1.18 + FastEmbed — five distinct Qdrant primitives are the product, not a sidecar.
  • Reason to exist: Claude Code silently migrates and deletes old sessions; Memex preserves a corpus you own, traversed by similarity not keywords.
  • Built for Qdrant Vector Space Day 2026 — "Think Outside the Bot."
  • Status: v0.1.0, macOS Apple Silicon, unsigned MVP — needs local Qdrant on localhost:6334.
