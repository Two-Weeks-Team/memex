The lesson in one line: that wasn't search

Associative recall · it comes to you

A context — your current task — calls up a related past on its own, by similarity, not keywords. You're the recipient.

  • The situation triggers retrieval, not your fingers.
  • Things you'd forgotten resurface because the moment evoked them.
  • It's Vannevar Bush's 1945 idea: follow associative trails, not a search box.

What Memex actually is

A non-chatbot tool with no LLM at runtime. It reads your Claude Code and Codex session history (~/.claude/projects/**/*.jsonl plus legacy transcripts), embeds it into a Qdrant vector space, and lets you move through it by similarity — so the right past can come back to you. It exists because Claude Code silently migrates and deletes old sessions; Memex keeps a corpus you own.

Topology & Lens

Move by similarity

A distance-matrix MST graph (Topology) and named-vector weighting (Lens) let you travel associative trails.

Time Machine & Mix

Replay & discovery

Replay any session in order; Mix uses positive/negative signals for discovery — plus Dashboard, Companion, Wrapped, Loop Breaker.

Built for agents, too

Memex exposes 11 MCP tools over stdio + HTTP, so Claude Code, Codex, and Cursor can recall from your corpus directly. A server / Docker variant runs Qdrant, the web UI, and the MCP server in one image. 

# add Memex to Claude Code as an MCP server
claude mcp add memex -- memex-mcp
# or run the all-in-one server variant (Qdrant + web + MCP)
docker run -p 8080:8080 memex/server

Commands illustrate the integration shape; see the repo README for exact, current invocations.
