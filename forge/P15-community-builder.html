Where this is real — and where it's the direction

Memex today runs local-first and single-user: it indexes the session history on your own machine and surfaces from your corpus. The team room above shows the same associative-recall principle applied to a shared corpus — the server / Docker variant runs Qdrant, the web UI, and MCP in one image, which is the foundation a team build sits on. We're showing the idea honestly, not claiming multiplayer is shipped.

How a memory comes back to you

No keyword search, no chatbot. Memex reads similarity, so a recollection rises because the moment evoked it.

01 · PRESERVE

The corpus you own

Claude Code silently migrates and deletes old sessions. Memex reads ~/.claude/projects/**/*.jsonl plus legacy transcripts and keeps a corpus that stays yours.

02 · EMBED

Distance, not keywords

Sessions become vectors in Qdrant. The space is navigable by similarity, so two moments that felt the same end up near each other — even with no shared words.

03 · SURFACE

Recall arrives unbidden

When your current context resembles a past one, proactive Recall brings the relevant memory forward on its own — the recipient experience you saw above.

Give your agents the same shared memory

Memex exposes 11 MCP tools over stdio + HTTP. It works in Claude Code, Codex, and Cursor, and the server / Docker variant bundles Qdrant, the web UI, and MCP in one image.

# add Memex as an MCP server to your agent claude mcp add memex -- memex serve --stdio # or run the all-in-one server variant (Qdrant + web + MCP) docker run -p 8080:8080 ghcr.io/two-weeks-team/memex

Built for Qdrant Vector Space Day 2026 — "Think Outside the Bot." Memex is a non-chatbot, LLM-free-at-runtime way to move through what your team already learned.
