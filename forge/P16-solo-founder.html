What Memex actually is

A non-chatbot, LLM-free-at-runtime tool over your own Claude Code / Codex session history — ~/.claude/projects/**/*.jsonl plus legacy transcripts and history.jsonl. It turns that history into a navigable spatial, associative memory backed by Qdrant. Built for Qdrant Vector Space Day 2026 — "Think Outside the Bot."

A corpus you own

Claude Code silently migrates and deletes old sessions. Memex preserves your history so it can't quietly vanish — and lets you move through it by similarity, not keywords.

Associative, not a search box

In the spirit of Vannevar Bush's 1945 Memex and its associative trails. The past arrives because the present situation evoked it — there's nothing for you to query.

One image, one operator

A server / Docker variant runs Qdrant + web + MCP in a single image. Stand it up once, alone, and forget it's there.

surfaces

  • Time Machine — replay a past session step by step.
  • Lens — named-vector weighted search across your corpus.
  • Topology — distance-matrix → MST graph of your work.
  • Mix / Discovery — positive/negative recommendation.
  • Proactive Recall — the past surfacing on its own (the demo above).
  • Dashboard + the new Companion / Wrapped / Loop Breaker agent-memory layer.

Wire it into your agent

11 MCP tools over stdio + HTTP. Works in Claude Code, Codex, and Cursor — so your past self can speak up inside the tool you already work in, without you switching context.

# add Memex as an MCP server to your agent
$ claude mcp add memex -- memex-mcp # stdio, local
✓ memex registered — 11 tools available

# or run the all-in-one server/Docker variant (Qdrant + web + MCP)
$ docker run --rm -p 6333:6333 -p 8080:8080 \
    -v ~/.claude:/data/claude:ro two-weeks-team/memex
