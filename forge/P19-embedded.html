Three moments fire on their own cadence — triggered by a situation, not by a keystroke. Memex is the recipient-not-seeker model, baked into the agent loop. All three are deterministic, sub-second on a ~70-session corpus, and run with zero LLM at runtime.

🧠

Cold Start Killer

fires when a new session opens

The agent boots with amnesia. Memex mines past sessions for this cwd and hands the agent a ready-to-inject primer — decisions, pitfalls, stack fingerprint — at turn 0.

🔁

Loop Breaker

fires after ≥3 tool errors in 10 turns

A background watcher notices you're stuck and surfaces the past session where past-you broke a similar loop — with a deep-link to the exact pivot turn.

🔔

Proactive Recall

fires on a fresh matching error

Hit a tool_result.is_error that matches a past error-vector neighbor above threshold? The session that solved it surfaces, unprompted.

Backed by Qdrant primitives (named-vector neighbors, payload-filtered scroll, Discovery API), all on localhost:6334. The MCP daemon never speaks to anything outside your machine.

Install & go — three lines

Point Claude Code at the same memex binary the desktop app already runs. One wiring step; every future session inherits the memory layer.

1

Start the load-bearing Qdrant

Memex stores your corpus as vectors in a local Qdrant. One command starts it.

# from the memex repo root
bash scripts/start-qdrant.sh
2

Wire the MCP server into Claude Code

One-time registration. Or run memex install-mcp --run to print & execute the exact line for your machine.

claude mcp add memex /path/to/memex/src-tauri/target/release/memex mcp

# verify
claude mcp list
#  memex: /…/memex mcp - ✓ Connected
3

Open a session — the memory arrives on its own

No further wiring. Optionally pre-warm a primer into your prompt by hand:

memex memory --cwd "$(pwd)" --limit 8 | pbcopy
# → ~1.5 KB markdown primer, ready to paste at turn 0

Prefer headless? The server variant runs Qdrant + web + MCP-over-HTTP in a single Docker image (POST /mcp) — no macOS app required.

11 MCP tools — typed handles on your own memory

Each maps directly to a Qdrant primitive that powers the desktop UI. Your agent calls them mid-conversation; the page below is just an example gallery — the product is the SDK surface.

find_similar_sessions

Five-vector Lens search over past sessions, with per-vector contribution scores in the response.

find_similar_error

Neighbor search on the error named vector, filtered to has_errors=true — the sessions that resolved it.

predict_next_action

"What would past-you do next?" — neighbor pivot-walk + tool-call aggregation by frequency × similarity.

mix_similar_sessions

Qdrant Discovery API — sessions near your positives, away from your negatives. Recommendation, not retrieval.

get_session_summary

Metadata payload: project, branch, ai_title, start/end, turn counts, has_errors.

get_session_turn

A single turn re-parsed from source jsonl — full text + tool calls + tool results.

list_recent_sessions

Most-recent-first walk of ~/.claude/projects — works even before Qdrant is warm.

analyze_corpus_topology

MST of session content vectors, per-project auto-labels, cross-project bridges, gap insights.

snapshot_export

Server-side snapshot of the entire collection to a portable .snapshot file.

get_project_memoryNEW

The Cold Start Killer. Returns a ready-to-inject memory primer (decisions, pitfalls, stack) as markdown for turn 0.

generate_wrapped_reportNEW

Engineering Wrapped — a corpus-wide digest for the last N days, screenshot-friendly markdown.

Why this is a layer, not an app

Claude Code rewrites its session storage every few months and silently deletes old files. Memex preserves a corpus you own in Qdrant — and exposes it through MCP so it follows your agent across every session, every client.
