On Associative Trails A working realization of Vannevar Bush's memex, eighty years on

Two-Weeks-Team · Memex project  |  correspondence: the repository, below

Abstract. In 1945, Vannevar Bush proposed the memex: a device that would store a person's records and let them be retrieved not by indexing but by association — chains of linked items he called associative trails.[1] This page describes a small, concrete tool that takes that idea literally for one specific corpus: your Claude Code / Codex session history. It is not a chatbot and runs no LLM at query time; it embeds your transcripts and stores them in Qdrant[3] so you can move through them by similarity rather than keywords. The claims here are deliberately narrow and verifiable against the source.

§1 The problem, stated plainly

A developer's working memory of a project lives partly in chat transcripts. Those transcripts accumulate on disk as ~/.claude/projects/**/*.jsonl (plus legacy transcripts and a history.jsonl). Tooling tends to treat them as disposable: sessions are silently migrated or deleted. The corpus you actually thought through is therefore both large and fragile, and it is keyed by nothing useful — you cannot grep your way back to "that approach we discarded last month," because you no longer remember the words you used.

Bush diagnosed exactly this gap. The trouble, he wrote, is not that we record too little but that retrieval is artificial:

"The human mind … operates by association. With one item in its grasp, it snaps instantly to the next that is suggested by the association of thoughts… Selection by association, rather than by indexing, may yet be mechanized." — V. Bush, As We May Think, The Atlantic, July 1945.[1]

§2 An associative trail, illustrated

The figure below is the page's one interactive element. It is not a search box. Each node is a session fragment; the lines are trails — proximity in embedding space. Move focus to (or hover) any node: its nearest neighbours surface, and the trail between them lights up. This is the same gesture the tool offers — approaching one memory brings the associated ones forward, the way a thought "comes back to you." Keyboard users: Tab through the nodes.

§3 Method — what the tool actually does

The mapping from Bush's 1945 specification to 2026 mechanics is direct, and we state it without embellishment.

Table 1. Bush's memex faculties and their concrete counterparts. No claim is made that this exhausts Bush's vision; only that these correspondences are literal.
Memex faculty (1945)Memex tool (2026)
A private store of "books, records, and communications"Local session corpus: ~/.claude/projects/**/*.jsonl + legacy transcripts + history.jsonl — a corpus you own.
Retrieval by association, not by indexVector similarity in Qdrant; movement by distance, not by keyword.
"Associative trails" of linked itemsTopology: distance-matrix → minimum-spanning-tree graph over fragments.
Trails the user can re-traverse laterTime Machine (replay) and Lens (named-vector weighted views).
Items that resurface when relevantProactive Recall and Mix / Discovery (positive / negative recommendation).

Design constraints (and what they rule out)

  • LLM-free at query time. Embeddings are computed and stored; retrieval is vector arithmetic. There is no generative model answering you, so there is nothing to hallucinate a citation. This is a deliberate limitation, not a feature claim.
  • No chatbot surface. The interfaces are spatial (Topology, Time Machine, Dashboard, and a Companion / Wrapped / Loop-Breaker agent-memory layer), not conversational.
  • Reproducible substrate. A server / Docker variant packages Qdrant + web UI + MCP server in one image, so the experiment can be re-run on another machine.

§4 Reproducibility & agent integration

The corpus, the store, and the access layer are all open. Memex exposes 11 MCP tools over stdio and HTTP, so an agent in Claude Code, Codex, or Cursor can traverse the same trails programmatically. To register it as an MCP server: 

# register the Memex MCP server with Claude Code
claude mcp add memex -- memex-mcp        # 11 tools over stdio
# or run the all-in-one image (Qdrant + web UI + MCP)
docker run -p 6333:6333 -p 8080:8080 memex:server

Source, the live landing page, and the server variant are linked below. We encourage inspection of the ingestion and embedding code rather than taking these descriptions on faith — the value of an associative memory is precisely that it should be auditable.

§5 Limitations & honest scope

  • Retrieval quality is bounded by the embedding model and by how much of your history survives on disk; Memex preserves the corpus but cannot recover sessions already deleted before first ingest.
  • "Surfacing" is similarity, not understanding. A neighbour in vector space is plausibly related, not provably so. Treat surfaced trails as leads, not conclusions.
  • This page is a project preview built for a hackathon. Numbers, latencies, and recall metrics are intentionally absent rather than estimated, because they have not been measured under a fixed protocol.
