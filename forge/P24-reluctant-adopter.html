For the reluctant adopter

You probably don't want another tool. Here's why you'd refuse this one — and the one reason that might change your mind.

No pitch yet. Let's start with the honest case against Memex. If none of these objections land, you're not the audience and that's fine.

  • ✕
    "It's another chatbot wrapper. I have enough of those." — Fair. So Memex runs no LLM at runtime and has no chat box.
  • ✕
    "I'd have to remember to use it." — That's the usual trap. Most memory tools make you do the recalling.
  • ✕
    "Search tools never find the thing I actually meant." — Keyword search doesn't. Memex moves by similarity, not exact words.
  • ✕
    "My session history is private. I'm not uploading it." — Correct stance. The corpus stays on your machine; you own it.

The objection that matters most is the second one. So here is the only claim worth making: you don't recall anything. The relevant past session arrives on its own — because the moment called it up. Don't take the word for it. Watch it happen below, without touching anything.

Disclosure

What Memex will not do

It won't answer questions in prose.

There is no chat. It returns the actual past session that matches your current context — you read the real thing, not a generated summary that might invent details.

It won't work on an empty history.

Recall surfaces your past. With no Claude Code / Codex sessions to index, there's nothing to rise unbidden. It earns its value over time.

It won't run with zero setup.

You build an index first (Qdrant + an embedding pass over ~/.claude/projects/**/*.jsonl). The server/Docker variant bundles Qdrant + web + MCP in one image to keep this short.

It won't guarantee the right memory every time.

Surfacing is by vector similarity. It's an associative trail, not an oracle — sometimes the thing it brings up is adjacent, not exact. We'd rather say that plainly.
