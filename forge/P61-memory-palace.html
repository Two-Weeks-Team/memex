The feeling

You didn't go looking. The room was already lit.

You don't journal every debugging session — yet the moment a familiar problem returns, the night you solved it last time comes back on its own. Memex is built around that reflex. It watches the session you already own and, when the current context lines up with something from your past, the relevant moment surfaces unbidden — the way a forgotten fact returns mid-task. No prompt. No query box. You're the recipient, not the seeker.

Concretely: Memex turns your Claude Code / Codex history into a navigable, spatial memory. Each past session is a room you can walk back into — and sometimes a door opens before you reach for it.

For your agent

Hand the corridor to Claude Code, Codex, or Cursor

Memex exposes 11 MCP tools over stdio + HTTP — each a typed handle on a Qdrant primitive the desktop app already uses. Wire it once and your agent can walk these rooms on its own. 

# one-time wiring — point Claude Code at the memex binary you already run
claude mcp add memex /path/to/memex/src-tauri/target/release/memex mcp
# or let memex print the line for you
memex install-mcp

What it reads

Your local session history — ~/.claude/projects/**/*.jsonl, legacy transcripts, and history.jsonl.

No LLM at runtime

Not a chatbot. Retrieval runs on vectors in Qdrant on localhost:6334; the corpus stays on your machine.

Headless option

The server / Docker variant runs Qdrant + web UI/API + MCP in one image.

Why it exists

Claude Code silently migrates and deletes old sessions. Memex keeps a corpus you own and lets you move through it by similarity.
