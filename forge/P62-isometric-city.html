The feeling

You didn't go searching the skyline. One window lit on its own.

You don't journal every debugging session — yet the moment a familiar problem comes back, the night you solved it last time returns on its own. Memex is built around that reflex. It treats the session history you already own as a city of past nights, and when the current context lines up with one of them, that window lights unbidden and the moment comes back into view. No prompt. No query box. You're the recipient, not the seeker.

Concretely: Memex turns your Claude Code / Codex history into a navigable, spatial memory. Each past session is a building you can walk back toward — and sometimes a window is already glowing before you reach for it.

For your agent

Hand the city to Claude Code, Codex, or Cursor

Memex exposes 11 MCP tools over stdio + HTTP — each a typed handle on a Qdrant primitive the desktop app already uses. Wire it once and your agent can walk these blocks on its own. 

# one-time wiring — point Claude Code at the memex binary you already run
claude mcp add memex /path/to/memex/src-tauri/target/release/memex mcp
# or let memex print the line for you
memex install-mcp

What it reads

Your local session history — ~/.claude/projects/**/*.jsonl, legacy transcripts, and history.jsonl.

No LLM at runtime

Not a chatbot. Retrieval runs on vectors in Qdrant on localhost:6334; the corpus stays on your machine.

Headless option

The server / Docker variant runs Qdrant + web UI/API + MCP in one image.

Why it exists

Claude Code silently migrates and deletes old sessions. Memex keeps a corpus you own and lets you move through it by similarity.
