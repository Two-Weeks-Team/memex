What Memex actually is

No magic claimed. Here is the plain shape of the thing.

The corpus
Your Claude Code / Codex session history (~/.claude/projects/**/*.jsonl), legacy transcripts, and history.jsonl — a record you own.
LLM-free at runtime
Recall runs on vector similarity, not a live model. No chatbot in the loop when a scene steps out.
Backed by Qdrant
History becomes a navigable spatial memory in a Qdrant vector store — you travel it by similarity, not keywords.
Why it exists
Claude Code silently migrates and deletes old sessions. Memex preserves the corpus and lets you return to it.
Built for
Qdrant Vector Space Day 2026 — "Think Outside the Bot."

It lives inside your agents

11 MCP tools over stdio + HTTP — works in Claude Code, Codex, and Cursor. A server / Docker variant runs Qdrant, the web UI, and MCP together in one image.

Add the memory layer to Claude Code:

# register the Memex MCP server (stdio)
claude mcp add memex -- memex-mcp

# or run the all-in-one server / Docker variant
# (Qdrant + web + MCP in a single image)

The 11 tools let an agent recall, replay, and connect across your own history — so the thread you had three weeks ago can step back into the light in the work you're doing now. See the repository for the tool list and the server variant.
