how recall works

It isn't a search box. The light does the remembering for you.

Vannevar Bush's 1945 Memex imagined memory you move through by association. Memex reads your own session history and lets the relevant moment gather into view — unbidden — when the situation evokes it.

— scattered across the floor —

Your history becomes the light

Claude Code & Codex transcripts — ~/.claude/projects/**/*.jsonl, legacy logs and history.jsonl — become a corpus you own, embedded into a vector space backed by Qdrant.

— the current ripples —

The moment sends a signal

Your present context — what you're debugging, designing, deciding — is matched by similarity against everything in the corpus. No query typed; the situation itself is the prompt.

— the light converges —

The scene gathers

The closest past moment converges out of the rippling light and holds — the way a forgotten fact returns mid-task — then ripples apart as the next moment calls another up.

honest about what it is

No chatbot. No LLM at runtime.

  • LLM-free at runtime. Retrieval is vector similarity over Qdrant — not a model generating answers.
  • A corpus you own. Claude Code silently migrates and deletes old sessions; Memex preserves yours.
  • Spatial, not keyword. You move through memory by similarity, the way Bush's Memex imagined.
  • Runs as a server too. A Docker variant bundles Qdrant + web + MCP in one image.

Works inside your agent

11 MCP tools over stdio + HTTP — usable from Claude Code, Codex, and Cursor, so a past memory can gather right where you're working.
