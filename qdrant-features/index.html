Why 12 deferred items? The "deferred" status doesn't mean "missed" — each non-adoption maps to one of 5 rationale categories. None are bugs; all are intentional decisions documented at the point of adoption (or non-adoption). This tab makes the criteria explicit so a reviewer can challenge a category rather than challenge 12 individual choices.

Promotion path: when a deferred item's category condition flips (use case emerges, scale changes, overlap dissolves, follow-up PR ships, architecture migrates), the row moves from §B "wired:off" or §C "not-wired" in docs/wired-but-dormant.md to §A "wired:on" — same mechanism that promoted RRF / MMR (#15) and added identifier_tokens (#14).
A Use case absent 4 items
Criterion: the feature solves a problem we don't have. Adopting now would be speculative.
FeatureWhy we don't need it
Geo filter We don't index "where I was when I wrote this". One payload-index line if the laptop-on-trip filtering ever becomes a feature.
TokenizerType::Whitespace The default Word tokenizer handles natural-language punctuation better. No specific use case for whitespace-only splitting in our corpus.
TokenizerType::Prefix No typeahead / search-as-you-type surface in current UX. Revisit if that surface lands.
UpdateVectors (swap a vector in place) index_session rebuilds the whole point on every change. UpdateVectors only buys speed when the vector itself changes more often than the payload — that's not our pattern.
B Scale mismatch 2 items
Criterion: the feature pays off at a corpus size or operational scale beyond hackathon scope. Adopting now is overengineering.
FeatureWhen it becomes worth adopting
Collection aliases (blue-green migration) Single-collection drop+rebuild is acceptable at our point count. Aliases land when v3 → v4 migration happens at 100k+ session corpora and zero-downtime matters.
Multi-collection (one per macro-topic) One memex_sessions_v3 has been enough. Cross-collection search lands in Qdrant 1.19; revisit when sharding by topic outperforms the single-collection HNSW.
C Functional overlap / redundant 2 items
Criterion: an already-adopted feature covers the same job. Adding the redundant feature would only expand surface area.
FeatureWhat covers it instead
Query::Recommend (AverageVector · BestScore) DiscoverInput + ContextInput already covers "more like this batch" with explicit positive/negative pairs (KB-03). Recommend is a redundant entry point. Revisit if DiscoverInput's target-required constraint becomes painful.
TokenizerType::Multilingual Our corpus is English session titles + identifiers. BGE-small embedding handles language semantics on the vector side; the multilingual tokenizer carries charabia / vaporetto overhead for non-Latin scripts we don't index.
D Deferred to follow-up PR 2 items
Criterion: the feature would be in scope, but the first-round delivery already spans the interesting space. Adding the rest is a small mechanical follow-up if needed.
FeatureWhat was prioritised first
Scalar (int8) quantization Issue #13 (PR #18) shipped f32 / tq-bits1 / tq-bits2 as the QuantMode triple — these span the interesting recall vs. disk-size space. Adding Scalar is a ~30-LOC variant in the same enum if a sweep needs more granularity.
Binary quantization Same rationale as Scalar. Same follow-up size if added.
E Different architectural layer 2 items
Criterion: the feature would handle a concern, but a different layer of the architecture already handles it more naturally. Adopting the Qdrant-side primitive would duplicate without adding value.
FeatureWhere it's actually handled
MCP write tools + memex_points_indexed_total MCP-side increment MCP is read-only by design (mcp.rs:3: "Qdrant primitives are hidden"). Adding write tools is a product/security decision (auth, schema, idempotency, abuse model). Issue #16 Stage 2 activates only when a future product PR introduces that surface. The always-zero counter is Prometheus-best-practice for cross-deployment metric uniformity.
API key / JWT / TLS auth on the embedded Qdrant Qdrant is loopback-bound to 127.0.0.1 in docker-compose.yml (THR-06). TLS termination + auth for the all-in-one server variant lives in the Caddy reverse proxy. Adopting Qdrant-side auth would be defence-in-depth at the cost of operator complexity, not a correctness gain.
