# P8 — E2E Validation Evidence

This directory holds **empirical proof artifacts** generated by P8 of the
SOTA v3.2 plan. The files were produced by running the two helper scripts
against the live local Qdrant 1.18 instance and the real 111-session
corpus (`~/.claude/projects` + `~/.codex/sessions`).

## How to regenerate

```bash
# 1) Start Qdrant (one-time, persists across runs)
docker run -d --name memex-qdrant -p 6333:6333 -p 6334:6334 \
  qdrant/qdrant:v1.18.0

# 2) Index the real corpus
./src-tauri/target/release/memex scan --index
# → indexed 110/111 session(s) into 'memex_sessions_v3'

# 3) Run the CLI smoke test (writes the *.json files)
bash scripts/demo/smoke-test.sh --json
# → exit 0; ✓ all 7 surfaces returned non-empty

# 4) Capture deep-link screenshots (requires installed Memex.app)
bash scripts/demo/capture-screenshots.sh
# → 5 PNGs in screenshots/
```

## Notes on screenshot capture

`capture-screenshots.sh` reads the Memex main window bounds via
`System Events` and uses `screencapture -R x,y,w,h` to capture only that
rectangle. This avoids the macOS-14+ regression where `screencapture -l`
silently falls back to full-screen.

Between routes the script presses **Escape** to close any open `<dialog>`
so each PNG captures the surface that the deep link actually opened
(rather than stacked modals from prior routes).

## Why these files are committed

Per the P8 `/goal` END STATE check #11: `ls tests/e2e/screenshots/*.png
→ >= 5 files`. The JSON files satisfy checks #2-#8 by providing
non-empty, reproducible CLI evidence that survives in git history.

The artifacts are also useful as fixtures for future regression
verification — if a surface starts returning empty in the wild, diff
against these baseline outputs.
